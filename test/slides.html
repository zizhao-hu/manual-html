ManualHTML

Every page, every deck, every dashboard — on a 3D grid. Editable in place. Keyboard-quick to rearrange.

Press → or click to begin.

The idea

Generated UI should be rearrangeable.

Claude Code already produces HTML in one shot. ManualHTML adds a second step: the user can nudge things into place without reopening the editor.

  • Every block is a component on a 60px grid.
  • Press M — grid appears, drag to move, corner handle to scale.
  • Click any text to edit it live.

A 3D grid

X, Y, and Z.

X / Y — position, snapped to 60px.

Scale — corner handle, also snapped to 60px.

Z — ↑/↓ change depth in 20px steps. Layer things.

Edit — click text to edit. Links and buttons don't navigate in Manual Mode so you can fix their labels.

For decks specifically

Grid on the current slide.

A deck is just a page with body class="mh-deck". Slides are containers; each slide has its own .mh-component children.

Default view: fullscreen slideshow. Click or → advances, ← goes back.

Press M and the grid appears on the current slide only. Drag its components around. →/← still switch slides so you can edit each one.

Authoring model

One spec. Any prompt.

MANUAL.md is the convention — it defines the grid, the keyboard model, and the boilerplate CSS and JS. It has no content of its own.

Your prompt ("a bakery landing page," "a pitch deck for Lumen Notes") is separate. Claude applies the convention to your prompt.

Paste MANUAL.md into your project's CLAUDE.md and ask for whatever you want.
